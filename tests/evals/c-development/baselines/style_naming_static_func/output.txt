Here's the internal helper added to src/my-utils-list.c. Static functions use the
_<module>_<action> naming pattern without the project prefix, since they're
file-scoped and not part of the public API.

```c
static void _list_flush_writes(my_utils_list_t* list) {
    if (!list->pending) return;
    my_utils_list_node_t* cur = list->write_buf;
    while (cur) {
        my_utils_list_node_t* next = cur->next;
        cur->next = list->head;
        list->head = cur;
        list->size++;
        cur = next;
    }
    list->write_buf = NULL;
    list->pending = 0;
}
```

The function is called internally before reads to ensure consistency. It follows
the static naming convention: underscore prefix + module name + action.
