# Build System

> Recommended trigger: auto-include when editing `CMakeLists.txt` / `*.cmake` files.

## Stack

- C11, CMake >= 3.16
- Output directory: `out/`

## CMake Rules

### Option Naming

All project-level options use `{PROJECT}_ENABLE_<FEATURE>`:

| Option | Default | Purpose |
|--------|---------|---------|
| `{PROJECT}_ENABLE_TESTING` | ON | Unit tests |
| `{PROJECT}_ENABLE_ASAN` | OFF | AddressSanitizer |
| `{PROJECT}_ENABLE_TSAN` | OFF | ThreadSanitizer |
| `{PROJECT}_ENABLE_UBSAN` | OFF | UndefinedBehaviorSanitizer |
| `{PROJECT}_ENABLE_COVERAGE` | OFF | Code coverage |
| `{PROJECT}_ENABLE_DYNAMIC_LIBRARY` | OFF | Shared lib (library projects only) |

Do not add options without the `{PROJECT}_ENABLE_` prefix.

### Adding Source Files

Add new `.c` files to the `SRCS` list in root `CMakeLists.txt`. Platform-specific sources go in the `WIN32` or `UNIX` conditional blocks:

```cmake
set(SRCS
    src/{project}-<module>.c
)
if(WIN32)
    list(APPEND SRCS src/platform/win/platform-<module>.c)
endif()
if(UNIX)
    list(APPEND SRCS src/platform/unix/platform-<module>.c)
endif()
```

### Adding Tests

Use the helper function in `tests/CMakeLists.txt`:

```cmake
{project}_add_test(<module>)
```

This creates executable `test-<module>` from `tests/test-<module>.c`, linked against `{project}`.

### Applying Sanitizers

Use the helper function from `cmake/{project}-utils.cmake`:

```cmake
{project}_apply_sanitizer(<target> {PROJECT}_ENABLE_ASAN address)
{project}_apply_sanitizer(<target> {PROJECT}_ENABLE_TSAN thread)
{project}_apply_sanitizer(<target> {PROJECT}_ENABLE_UBSAN undefined)
```

Do not add sanitizer flags manually with `target_compile_options`.

### Prohibited Patterns

- No hardcoded absolute paths
- No `find_package` without version constraints
- No `CMAKE_C_FLAGS` modification — use `target_compile_options` on specific targets
- No `link_directories` — use `target_link_libraries` with full paths or imported targets

## Commands

### Build

```bash
cmake -B out
cmake --build out
```

### Test

```bash
# Windows (multi-config generator)
ctest --test-dir out -C Debug --output-on-failure

# Linux/macOS
ctest --test-dir out --output-on-failure
```

### Sanitizer Build

```bash
# AddressSanitizer
cmake -B out -D{PROJECT}_ENABLE_ASAN=ON
cmake --build out

# ThreadSanitizer
cmake -B out -D{PROJECT}_ENABLE_TSAN=ON
cmake --build out

# UndefinedBehaviorSanitizer
cmake -B out -D{PROJECT}_ENABLE_UBSAN=ON
cmake --build out
```

ASAN and TSAN cannot be enabled simultaneously.

### Coverage (Linux/macOS only)

Requires: `lcov`, `genhtml`. Testing must be enabled (`{PROJECT}_ENABLE_TESTING=ON`, which is the default).

```bash
cmake -B out -D{PROJECT}_ENABLE_COVERAGE=ON -D{PROJECT}_ENABLE_TESTING=ON
cmake --build out
ctest --test-dir out --output-on-failure
```

Coverage report is generated by the `coverage` target in `tests/CMakeLists.txt`:

```bash
cmake --build out --target coverage
# Report: out/coverage/index.html
```

### Coverage (Windows)

Requires: OpenCppCoverage. Testing must be enabled.

```bash
cmake -B out -D{PROJECT}_ENABLE_COVERAGE=ON -D{PROJECT}_ENABLE_TESTING=ON
cmake --build out --config Debug
cmake --build out --target coverage
# Report: out/coverage/index.html
```
